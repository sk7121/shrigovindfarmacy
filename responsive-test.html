🔍 Detected Issues (Need Manual Verification)

1. Modal Bottom Sheet - iPhone Safe Area

Modal bottom sheet on mobile may not have padding for iPhone home indicator bar. Check if modal content gets cut off at the bottom on devices with home bars.

Fix Location: style.css around line 3230-3245

2. Doctor CTA Badge Positioning

The .doc-cta-badge uses absolute positioning but parent .doctor-image-stack may not have position: relative, causing the badge to appear in wrong location.

Fix Location: style.css around line 2270

3. Toast & WhatsApp Float Overlap on Mobile

On cart page, toast notifications at bottom: 110px might overlap with WhatsApp float at bottom: 90px on small screens.

Fix Location: cart.css lines 720-728

4. Dropdown Menus on Touch Tablets

Category dropdowns rely on hover which doesn't work on touch devices between 768px-1024px. Users may not be able to access dropdown menu items.

Fix Location: style.css around line 355-375

5. Category Sticky Navigation

Sticky nav top: 70px may collide with header if announcement bar is visible (36px extra height).

Fix Location: categories.css line 140

6. Modal Z-Index vs Mobile Menu

Modal backdrop (z-index: 890) sits below mobile menu (z-index: 900), causing stacking issues when both are open.

Fix Location: style.css lines 910-920

7. Slider Content Clipping at 320px

Hero slider at 320px width has only 240px height. Content with heading, badge, buttons, and stats may overflow and get clipped by overflow: hidden.

Fix Location: style.css line 1130

8. Duplicate Z-Indexes Across CSS Files

Multiple CSS files define .wa-float (9998) and .toast-zone (9999). If loaded together, last file wins causing inconsistent behavior.

Fix Location: All CSS files (various locations)
